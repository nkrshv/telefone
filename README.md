# telefone

Защищённые P2P видеозвонки в браузере. MVP: «только звонки» (1:1), со сквозным
шифрованием (E2EE).

## Что внутри

- **P2P WebRTC** — аудио/видео идёт напрямую между участниками, не через сервер.
- **DTLS-SRTP** — стандартное транспортное шифрование WebRTC (всегда включено).
- **Дополнительный слой E2EE** через WebRTC Insertable Streams: каждый кадр
  шифруется **AES-256-GCM** ключом, выведенным (HKDF-SHA256) из секрета комнаты.
  Секрет передаётся только между участниками и **никогда не попадает на
  сигналинг-сервер**.
- **Код безопасности** (отпечаток ключа) для голосовой сверки — защита от MITM.
- **Сигналинг** — PeerJS (публичный PeerServer). Сервер видит только временный
  peer-id и обмен SDP/ICE, но не медиа и не ключи.

## Модель «код комнаты»

Код комнаты имеет вид `peerId.secret`:

- `peerId` — идентификатор в сигналинг-сети (виден серверу);
- `secret` — высокоэнтропийный секрет (24 байта), из которого выводится ключ
  AES-GCM. На сервер не уходит.

Хост создаёт звонок и получает код/ссылку; гость вставляет код и подключается.

## Технологии

Vite + React + TypeScript, статический сайт без бэкенда. `peerjs` для сигналинга,
WebCrypto + Insertable Streams для шифрования.

## Запуск

```bash
npm install
npm run dev      # http://localhost:5173
npm run build    # сборка в dist/
npm run lint
```

> Для камеры/микрофона нужен `https://` (или `localhost`). Доп. слой E2EE
> (Insertable Streams) работает в Chrome/Edge; в браузерах без поддержки звонок
> идёт по DTLS-SRTP с предупреждением в интерфейсе.

## Ограничения MVP / дальше

- Нет TURN-сервера — при «строгом» (симметричном) NAT звонок может не
  установиться. Следующий шаг — добавить TURN (coturn).
- Публичный сигналинг-сервер PeerJS — для прод стоит поднять свой.
- Только 1:1, без аккаунтов и чата.
